# 18. ROADMAP.md

## Version 1.0: Professional Local AI Desktop

### Goal

Deliver a reliable desktop AI workspace with chat, memory, local models, project context, basic coding assistance, and secure settings.

| Area | Features |
|---|---|
| Desktop | Cross-platform app shell, secure local backend, settings, updates. |
| Chat | Conversations, streaming, history, branches, attachments. |
| Models | Ollama, cloud model adapters, model selector, fallback routing. |
| Memory | User memory, conversation memory, project memory, inspect/edit/delete. |
| Workspace | File explorer, project setup, indexing, semantic search. |
| Coding | Explain, review, generate, test suggestions. |
| Security | Secret storage, local-first privacy, audit logs, permission prompts. |
| UX | Dashboard, sidebar, command palette, themes, accessibility baseline. |

## Version 2.0: Creative, Agentic, and Extensible

### Goal

Expand AAZHI AI into a creative and automated AI platform with image studio, voice, supervised agents, and official plugins.

| Area | Features |
|---|---|
| Image | Text-to-image, image editing, variations, asset library. |
| Voice | Speech input, spoken responses, voice sessions. |
| Agents | Goal planning, supervised execution, approvals, task memory. |
| Plugins | Plugin SDK, sandbox runtime, official GitHub/Docker/Browser/Figma plugins. |
| Prompting | Prompt library, reusable templates, prompt evaluations. |
| Model Manager | Benchmarks, capability matrix, provider health, cost controls. |
| UX | Agent activity, image studio, plugin center, memory dashboard improvements. |

## Version 3.0: Team, Marketplace, and Enterprise Platform

### Goal

Scale AAZHI AI for teams, enterprises, plugin developers, and long-term ecosystem growth.

| Area | Features |
|---|---|
| Cloud sync | Account system, encrypted sync, device management. |
| Collaboration | Shared workspaces, team memory, comments, roles. |
| Enterprise | SSO, RBAC, audit export, retention policies, admin console. |
| Marketplace | Signed plugin marketplace, publisher portal, enterprise registry. |
| Agents | Multi-agent workflows, scheduled tasks, reusable automations. |
| Knowledge | Team knowledge graph, advanced citations, document intelligence. |
| Deployment | Private cloud, on-prem options, policy-managed model providers. |

## Roadmap Principles

| Principle | Description |
|---|---|
| Trust before autonomy | Agents and plugins should become powerful only after permission systems are mature. |
| Local before cloud dependency | Core value should not require a cloud account. |
| Platform before marketplace | Build stable plugin APIs internally before public marketplace launch. |
| Memory with control | Memory must remain visible, editable, and deleteable at every stage. |
